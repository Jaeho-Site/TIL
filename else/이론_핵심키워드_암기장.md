## 1. 응집도 / 결합도 ★★★ (사실상 매회, 25-1 결합도 → 26-1 응집도 → 다음은 결합도 차례)

**응집도 (높을수록 좋음) 높→낮 : `기·순·교·절·시·논·우`**

| 시그널 | 답 |
|---|---|
| 하나의 기능만 밀접하게 | **기능적** |
| 한 요소의 "출력"이 다음 요소의 "입력" | **순차적** |
| 서로 다른 기능이지만 "동일한 입출력" 사용 | **교환(통신)적** |
| 기능적 연관은 없으나 "순서대로 실행" | **절차적** |
| 같은 시간대에 처리 (초기화 등) | **시간적** |
| 유사한 성격끼리 묶임 (하나의 모듈에서 case로 분기) | **논리적** |
| 아무 관련 없음 | **우연적** |

**결합도 (낮을수록 좋음) 낮→높 : `자·스·제·외·공·내`**

| 시그널 | 답 |
|---|---|
| 단순 값(파라미터)만 전달 | **자료** |

| 외부의 다른 모듈에서 선언한 데이터 참조 | **외부** |
| "전역 변수" 공유·갱신 | **공통** |
| 다른 모듈의 "내부 기능·자료를 직접 참조/수정" | **내용** |

**서브넷 계산 공식 (2~3년마다 계산 문제):**
- CIDR `/n` = 서브넷 마스크에서 **1의 개수**
- 네트워크 주소 = IP **AND** 마스크 (22-2·25-2 "128, 62", 26-1 "/23")
- 호스트 수 = 2^(0의 개수) − 2 / 브로드캐스트 = 호스트부 전부 1 (25-1)
- OSI 7계층: **물·데·네·전·세·표·응** (데이터링크 21-3)
