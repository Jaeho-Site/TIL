데이터 규모 (N),허용 시간 복잡도,주요 알고리즘 예시

"N≤100",        O(N4),      "플로이드-워셜, 4중 for문"

"N≤500",        O(N3),      "단순한 3중 for문, 복잡한 동적 계획법(DP)"

"N≤2,000",      O(N2),      "이중 for문, 삽입/거품 정렬, 일반적인 DP"

"N≤100,000",    O(NlogN),   "퀵/병합/힙 정렬, 이진 탐색, 우선순위 큐(Heap)"

"N≤10,000,000", O(N),       "단순 for문, 그리디(Greedy), 투 포인터, 슬라이딩 윈도우"

"N>10,000,000", O(logN),    "이진 탐색 (Binary Search)"
