# 식별자 정리

### 통신을 가능하게 하는 3가지 핵심 식별자

데이터가 출발지 장비에서 목적지 Host에 도달하기까지, 이때 사용되는 3가지 필수적인 식별자는 **MAC 주소, IP 주소, Port 번호**이다. 이들은 서로 다른 계층에서 각기 다른 범위를 식별하는 역할을 담당한다.

**1. L2 (Data Link Layer)의 식별자 : MAC 주소 (하드웨어/NIC 식별자)**
MAC(Media Access Control) 주소는 데이터 링크 계층에서 사용되는 약 48bit(6byte) 크기의 물리적 식별자다. 네트워크 통신을 위해 컴퓨터나 스마트폰에 장착되는 네트워크 인터페이스 카드(NIC, 통상 랜카드)가 공장에서 제조될 때 하드웨어 칩셋에 영구적으로 각인되는 고유한 번호로, 원칙적으로 전 세계에서 유일무이한 값을 가진다. (재밌는점은 MAC주소는 변경이 가능하다.)
IP 주소가 논리적인 주소라면, MAC 주소는 물리적으로 직접 연결된 동일한 로컬 네트워크(LAN) 내에서 장비들을 실질적으로 구분하고, 데이터가 '바로 다음 목적지(Next Hop)' 장비로 이동하기 위한 물리적 이정표 역할을 한다. 데이터가 여러 라우터를 거쳐 광역 네트워크(WAN)를 횡단할 때, 출발지와 최종 목적지의 IP 주소는 변하지 않지만, 각 라우터 구간을 넘어갈 때마다 출발지 MAC 주소와 목적지 MAC 주소는 해당 구간의 장비들에 맞춰 끊임없이 갱신된다.

**2. L3 (Network Layer)의 식별자: IP 주소 (호스트/PC 식별자)**
IP(Internet Protocol) 주소는 네트워크 계층에서 사용되는 논리적 식별자로, 수십억 대의 컴퓨터가 연결된 거대한 인터넷 공간에서 특정 호스트(PC, 서버, 모바일 기기 등 인터넷에 연결된 컴퓨터를 Host라고함)를 유일하게 식별하기 위해 부여된다. 현재 널리 쓰이는 IPv4 체계에서는 32비트로 구성되며, 주소 고갈 문제를 해결하기 위해 도입된 IPv6 체계에서는 128비트로 구성된다.
IP 주소는 택배 시스템의 '도로명 주소'나 '우편번호'와 같이 네트워크 공간에서의 계층적인 위치 정보(네트워크 ID와 호스트 ID)를 내포하고 있다. 전 세계의 네트워크 노드에 위치한 라우터(Router) 장비들은 이 IP 주소의 네트워크 대역 정보를 라우팅 테이블과 대조 분석하여, 패킷이 목적지로 향하기 위해 통과해야 할 최적의 경로를 결정(Routing)하고 데이터를 포워딩한다. IP 주소라는 전역적인 식별자가 존재하기에, 데이터는 복잡한 경로 속에서도 길을 잃지 않고 수신측 호스트의 문 앞까지 정확하게 배달될 수 있다.

**3. L4 (Transport Layer)의 식별자: Port 번호 (프로세스 식별자)**
IP 주소를 통해 목적지 호스트(PC)까지 데이터가 도착했다 하더라도, 통신이 완전히 종료된 것은 아니다. 현대의 운영체제는 멀티태스킹 환경을 지원하므로, 하나의 컴퓨터 안에서는 웹 브라우저, 카카오톡 메신저, 온라인 게임, 이메일 클라이언트, 백그라운드 업데이트 프로세스 등 수십 개의 네트워크 응용 프로그램(프로세스)이 동시에 실행되며 각자의 데이터를 기다리고 있다. 만약 식별자가 IP 주소뿐이라면, 컴퓨터는 방금 수신한 데이터 패킷이 웹 브라우저가 화면에 렌더링해야 할 HTML 데이터인지, 아니면 카카오톡이 화면에 띄워야 할 메시지 데이터인지 구분할 방도가 없다.
이러한 문제를 해결하기 위해 전송 계층에서 도입한 논리적 식별자가 바로 Port 번호다. Port 번호는 운영체제 내에서 실행 중인 프로세스를 명확히 식별하는 16비트 숫자로, 외부에서 들어온 데이터가 정확한 애플리케이션의 메모리 공간으로 인계되도록 보장하는 최종 목적지 역할을 수행한다.
