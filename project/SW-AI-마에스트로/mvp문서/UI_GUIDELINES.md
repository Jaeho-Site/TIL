## 1. 비주얼 방향 (무드)

**키워드:** 자연 · 따뜻함 · 친근함 · 차분함 · 야외 · 신뢰 · 미니멀.

산책온은 **밝은 아침 산책길의 친근한 동네 가이드**처럼 느껴져야 한다 — 피트니스 트래커(차갑고, 데이터 과다)도, 보상 게임(요란하고, 채도 높음)도, 내비 앱(실용 일변도)도 아니다. *웰니스 앱의 친근함*에 *지도 앱의 명료함*을 더한 느낌을 생각하라.

- **여유롭고 차분하게:** 넉넉한 여백, 화면당 하나의 명확한 초점(P2).
- **따뜻한 자연 팔레트:** 나뭇잎의 초록 + 따뜻한 오프화이트 종이 + 햇살 어린 앰버 강조색.
- **크고, 부드럽고, 탭하기 쉽게:** 큼직한 둥근 카드, 큰 타이포, 엄지 친화적 타깃(P9).
- **지도 중심:** 지도는 1급 시민이며, 팔레트에 맞춰 스타일링(기본 타일 그대로가 아니라).
- **정직하고 조용한 UI:** 신뢰도와 주의 사항을 담백하게 표시; 다크 패턴 금지(P7, P8).

**참고 느낌:** Airbnb의 부드러움과 카드 언어, Apple의 차분한 타이포그래피 명료함, 웰니스 앱의 친근한 따뜻함 — 야외, 초록, 한국 도시 맥락에 맞게 각색.

## 3. 타이포그래피

- **폰트 패밀리:** 성능과 한글 지원을 위해 시스템 우선 — `-apple-system, "SF Pro", "Pretendard", "Noto Sans KR", Roboto, sans-serif`. 한글 텍스트에는 **Pretendard/Noto Sans KR** 선호. 둥글고 친근한 산세리프; 압축형이나 극단적 지오메트릭 서체는 피할 것.
- **스케일 (모바일, 16px 기준 rem):**

| 역할 | 크기 | 굵기 | 행간 |
|---|---|---|---|
| 디스플레이 (스플래시, 큰 숫자) | 2.25rem / 36px | 700 | 1.15 |
| H1 (화면 제목) | 1.75rem / 28px | 700 | 1.2 |
| H2 (섹션) | 1.375rem / 22px | 600 | 1.25 |
| H3 (카드 제목) | 1.125rem / 18px | 600 | 1.3 |
| 본문 | 1rem / 16px | 400 | 1.5 |
| 본문-강조 | 1rem / 16px | 600 | 1.5 |
| 캡션 / 메타 | 0.875rem / 14px | 400–500 | 1.4 |
| 마이크로 (칩, 라벨) | 0.75rem / 12px | 500 | 1.3 |

- **규칙:** 크고 읽기 쉬운 기본값(의미 있는 텍스트는 14px 미만 금지); 야외에서 한눈에 보기 위한 넉넉한 행간; 숫자(거리/시간/연속 기록)는 약간 더 크고 굵게(가능하면 등폭 숫자).

## 4. 간격, 라운드, 그림자

### 4.1 간격 스케일 (4px 기준)
`4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 · 48` (px). 기본 화면 패딩: **20px**. 카드 간 기본 간격: **16px**. 편안하고 여유롭게 — 빽빽하지 않게.

### 4.2 라운드 (부드럽고 둥글게)
| 토큰 | 값 | 용도 |
|---|---|---|
| `--radius-sm` | 8px | 칩, 작은 컨트롤 |
| `--radius-md` | 16px | 버튼, 입력 |
| `--radius-lg` | 20px | 카드 |
| `--radius-xl` | 24px | 바텀 시트(상단 모서리), 히어로 카드 |
| `--radius-pill` | 999px | 알약형 버튼, 태그 칩 |

### 4.3 그림자 (부드럽고 저대비)
| 레벨 | 그림자 |
|---|---|
| 카드 | `0 2px 8px rgba(28,43,36,0.06)` |
| 부상(주 카드, FAB) | `0 6px 20px rgba(28,43,36,0.10)` |
| 시트 | `0 -8px 28px rgba(28,43,36,0.14)` |

딱딱한 테두리보다 **부드러운 그림자 + 틴트 채움**을 선호하라. `--hairline`은 절제해서 사용.

## 5. 코어 컴포넌트

### 5.1 버튼
- **주(Primary):** `--brand-forest` 채움, 흰 텍스트, `--radius-md`(또는 CTA는 알약형), 최소 높이 **52px**, 주요 동작은 모바일에서 전체 너비("산책 시작", "추천 보기"). 눌림 → `--brand-forest-strong`.
- **보조(Secondary):** 아웃라인 또는 `--surface-alt` 채움, `--ink` 텍스트(예: "다른 코스 보기").
- **3차 / 텍스트:** 텍스트만, `--brand-forest` 라벨("게스트로 계속하기").
- **파괴적(Destructive):** `--danger` 텍스트/채움(데이터 삭제, 로그아웃 확인).
- 큰 탭 타깃(≥48px), 명확한 눌림/비활성/로딩 상태.

### 5.2 코스 카드 (시그니처 컴포넌트)
홈, 추천, 탐험, 저장 목록에서 사용.
- 둥근 `--radius-lg`, `--surface`, 부드러운 카드 그림자.
- **지도 썸네일**(경로는 `--brand-forest`로 그림)을 상단 또는 좌측에 히어로로.
- 제목(H3) + 메타 행: 거리 · 예상 시간 · 난이도.
- **목적 태그 칩**(§5.4) + **신뢰도 칩**(§5.5).
- 홈/추천의 주 카드에서는: 부상된 그림자, 더 크게, 아래에 이유 줄 포함.

### 5.3 이유 블록
- 구별되는 부드러운 컨테이너(`--brand-forest-tint` 배경, `--radius-md`), 작은 "AI" 또는 반짝임 글리프, 근거 있는 자연어 설명 1~2문장. 절대 일반적인 채움말 금지.

### 5.4 태그 칩 (목적)
- 알약형, 목적 컬러의 틴트 배경, 아이콘 + 라벨, `--radius-pill`. 선택 상태 = 목적 컬러 채움 + 흰색; 미선택 = 틴트 + 컬러 라벨.

### 5.5 신뢰도 칩 / 표시
- §2.3 컬러를 사용한 알약형 또는 작은 배지, **아이콘 + 단어**("검증됨", "의심", "미검증", "위험"). 지도에서는 해당 구간을 그에 맞게 색칠(위험 구간은 `--danger`, 점선).

### 5.6 피드백 컨트롤
- **원탭 평가:** 크고 친근한 버튼 두 개 — 👍(forest 틴트) / 👎(뉴트럴/앰버 틴트) — 엄지가 닿는, ≥64px 타깃.
- **태그 시트:** 아이콘+라벨 알약 칩 그리드; 다중 선택; 언제나 건너뛰기 가능. 최저 마찰 느낌(P4).

### 5.7 바텀 시트
- 둥근 상단 `--radius-xl`, 그랩 핸들, `--surface`, 시트 그림자. 조건(05), 문제 신고(09), 태그 시트(10)에 사용. 산책이 계속되어야 하는 곳에서는 비모달(P9); 닫기 가능.

### 5.8 하단 탭 바
- 4개 탭(홈 · 탐험 · 히스토리 · 프로필), 아이콘 + 라벨, 활성 = `--brand-forest`, 비활성 = `--ink-faint`. 은은한 상단 헤어라인으로 부상; 세이프 에어리어 대응.

### 5.9 통계 / 연속 기록 위젯
- 큰 숫자(디스플레이/H1) + 라벨; 연속 기록과 축하 순간에는 `--accent-sun` 사용. 차분하게 유지 — 축하는 따뜻하되 요란하지 않게(P8, 중독 패턴 없음).

## 6. 지도 스타일링
지도는 핵심이다; 기본 타일 그대로 내보내지 말고 팔레트에 맞춰 스타일링하라.
- **베이스:** `--bg-paper`에 맞춘 채도를 낮춘 따뜻한 밝은 베이스맵; 공원은 은은한 초록, 물은 부드러운 파랑.
- **경로 선:** `--brand-forest`, 5~6px, 둥근 끝; 추천 시 그려지는 애니메이션은 좋은 디테일(은은하게 유지).
- **핵심 지점:** 출발(forest 핀), 경유지(작은 점), 반환점(링), 도착(깃발). 구별되고 라벨 표기.
- **사용자 위치:** 명확한 "블루 닷" 상당물(`--info` 사용), 정확도 헤일로 포함(GPS 정직성 강화, P9).
- **구간:** 신뢰도별 색상(§2.3); 확정된 위험 = `--danger` 점선 + 주의 글리프.
- **문제 핀:** 의심은 앰버, 확정은 레드, 정확한 GPS 위치에 드롭.

## 7. 아이코노그래피 & 일러스트레이션
- **아이콘:** 둥글고, 일관된 획(예: ~2px), 친근함 — 부드러운 라운드 언어에 맞춤. 전체적으로 한 패밀리. 목적 아이콘(잎/방패/파동/발자국)은 브랜드 아이덴티티의 일부.
- **일러스트레이션:** 온보딩, 빈 상태, 축하를 위한 가볍고 따뜻한 자연 테마 스팟 일러스트 — 부드러운 그라디언트의 플랫, 초록 + 앰버. 무거운 3D나 마스코트 과다 스타일은 피할 것(추후 브랜드 마스코트가 정의되기 전까지).
- **이모지:** 제품은 이미 목적/태그에 이모지를 활용한다(🌳🤫🦵🛡️🚧🔇🚫🌑⚠️) — 따뜻함과 빠른 인지를 위해 텍스트와 함께 유지.

## 8. 모션 & 피드백
- **원칙:** 모션은 확인하고 잠깐 기쁨을 주되; 산책을 지연시키지 않는다(P1, P9).
- **지속 시간:** 마이크로 120ms, 표준 200~250ms, 시트 280~320ms; 이징 `cubic-bezier(0.2, 0.8, 0.2, 1)`(부드러운 ease-out).
- **시그니처 순간:** 추천 시 경로 그려짐; 산책 완료 시 부드러운 체크 + 연속 기록 상승; 배지 획득(따뜻하고, 짧고, 닫기 가능).
- **로딩:** 카드/지도는 스켈레톤(스피너 아님)으로 ~3초 추천이 즉각적으로 느껴지게; 연산이 길어지면 진행 표시.
- **절제:** 튀어오르는 잔소리, 주의를 끄는 반복, 가짜 긴박감 없음(P8).

## 9. 상태 (모두 디자인하라)
각 화면마다 디자인: **로딩**(스켈레톤), **빈 상태**(격려하며 명확한 다음 동작 포함 — 예: 첫 산책 유도), **에러**(쉬운 말 + 재시도, 절대 막다른 길 아님), **권한**(위치 거부 → 수동 경로), **오프라인**(캐시된 콘텐츠 + 정직한 안내), **얇은 데이터/저신뢰**(정직한 유의점, P7). 빈 상태와 에러 상태는 따뜻한 일러스트 + 단일 주 동작 사용.

**어드민 대시보드(A1–A3):** 별도의 데스크톱 웹 레이아웃은 모바일 앱보다 **더 조밀하고 중립적**으로 갈 수 있다 — 더 작은 라운드(8~12px), 표, `--surface-alt` 배경, 일관성을 위해 목적/신뢰도 팔레트를 쓴 데이터 시각화. 두 제품이 하나의 시스템으로 읽히도록 동일한 컬러 토큰을 유지하라.

## 10. 해야 할 것 / 하지 말 것 (비주얼 가드레일)
- **하라** 화면당 하나의 명확한 초점 동작 유지; 지도/코스를 앞세우고, 그다음 이유.
- **하라** 모든 색상 의미를 아이콘 + 라벨과 짝지어라(목적, 신뢰도).
- **하라** 큰 타깃, 큰 타이포, 햇빛을 위한 높은 대비 유지.
- **하라** 신뢰도를 정직하게 표시("미검증/낮음" 포함).
- **하지 마라** 요란하고 채도 높은 게이미피케이션 비주얼이나 가짜 긴박감 애니메이션.
- **하지 마라** 모바일 핵심 루프에 조밀한 대시보드나 긴 옵션 목록 제시.
- **하지 마라** 따뜻한 팔레트와 충돌하는 기본 지도 타일 그대로 사용.
- **하지 마라** 색상에만 의존하거나, 의미 있는 텍스트를 14px 미만으로 사용.
