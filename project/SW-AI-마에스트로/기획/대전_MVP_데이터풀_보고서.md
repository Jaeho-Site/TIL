# 산책온 MVP — 대전 단일 서비스 데이터풀 보고서

> 작성 기준: 2026-07-05 · 전제: **MVP를 대전광역시 한정으로만 출시**한다고 가정하고, 기존 확보 데이터를 "대전 위주로만" 재구성했을 때 실제 데이터풀 규모를 목적별로 구체화.
> 근거: API 검증 1·2차 보고서 + 국가소음 CSV 실측(대전 필터링) + 공공데이터포털 대전 데이터셋 조회.
> 표기 — **[실측]** 이번에 직접 다운로드/카운트 · **[조회]** 데이터셋 페이지 확인 · **[추론]** 미검증 추정.

---

## 0. 한 줄 결론

대전만 놓고 보면 **방범(조명·CCTV)과 보행 경로 데이터는 매우 두껍고**, **자연친화(대전둘레산길)는 3도시 중 유일하게 확보**되어 있으나, **조용함(소음)은 관측점이 6곳뿐으로 가장 얇다.** 즉 대전 MVP의 강점은 **안전·자연**, 보완 필요는 **소음·계단**이다.

## 2. 목적별 상세

### 🛡️ 방범 — 대전의 가장 두꺼운 풀 (강점)

**① 가로등(야간 조도) — 43,082건, 위경도 포함 [조회]**
- 데이터셋: `대전광역시_가로등 현황`(data.go.kr/15110054), 16개 컬럼(관리번호·행정읍면동·등주높이·광원종류·**위도·경도**).
- 4.3만 포인트는 블록 단위로 "밤에 밝은 길 / 어두운 길"을 실제로 스코어링할 수 있는 밀도. → **야간 안전 지수의 1차 재료로 충분.**
- 한계: 갱신 "수시(1회성, 2022-12 기준)" → 최신성은 떨어짐.

**② CCTV — 시·구 다층 커버 [조회]**
- 시 단위: `대전광역시_방범용 CCTV`(15109459, **실시간**), `차량방범용 CCTV`(15109478), `교통관제용 CCTV`(15109442) — 모두 위치·운영상태 API.
- 구 단위 보강: 서구 생활방범/교통단속/재난재해(15095265·15095267 등), 유성구 설치현황(15108837).
- → 시+구를 합치면 "CCTV 있는 구간" 태깅 가능. 정확한 총 대수는 데이터셋별 카운트 필요(개별 API 조회 시 확정).

**③ 보행자 안전 통계·시설**
- 대전 횡단보도조회서비스, 보행자 교통사고 통계(1차 PDF 인용: **대전 사고의 52%가 보행자, 55%가 교차로 횡단 중**) → 위험 교차로 회피 가중치.
- 전국보안등정보표준데이터(15017320)에 대전분 별도 존재.

### 🚶 보도/보행 — 필드 완전, 경로 무제한 (강점)

- **전국보행자전용도로 표준데이터(대전분)**: 컬럼이 목적에 정확히 부합 — `roadBt`(보도폭), `vidoInstallationCo`(CCTV 수), `lmpInstallationCo`(보안등 수), 보차분리여부, 점자블록·횡단보도·방호울타리·속도저감시설. **시종점 위경도 포함.** → 한 데이터셋에서 방범+보도 지수를 동시 산출.
  - 단, 현재 국가숲길 키로는 미등록 → **대전 대상 활용신청 1건**이면 발급.
- **TMAP 보행자 경로**: 대전에서 실호출 성공. **육교(turnType 125)를 대전 계룡로에서 실증.** 경로는 좌표·거리·시간·시설(turnType) 태깅과 함께 **무제한 생성** → 보도 데이터의 "선(line)"을 채우는 엔진.

### 🌳 자연친화 — 3도시 중 대전만 보유 (핵심 차별)

- **대전둘레산길**: 산림청 **국가숲길 지정**(2023). **12구간·총 약 138km**(1구간 보문산길 9.3km, 6구간 계족산 13.5km, 10구간 8.98km 등). 서울·부산은 국가숲길 0곳.
- 형태: 국가숲길 코스 GPX(조밀한 트랙포인트)로 확보 가능. POI API(15108060)는 현재 **제공기관 502 장애** → 복구 후 POI 상세 수집(장애는 대전만의 문제가 아니라 전국 공통).
- 도심 산책로 보강: `대전_자전거도로 및 산책로현황`, `대전광역시_자전거도로조회서비스 API`(15110708)로 둘레산길 밖 평지 코스 커버.

### 🦵 관절보호 — 부분 확보 (보완 필요)

- **강점**: TMAP `turnType`으로 육교(125)·지하보도(126)·경사로(128)·엘리베이터(218) 태깅 가능(육교는 대전 실증). 둘레산길 산악 구간의 경사 자체가 "관절 부담 코스" 신호.
- **공백**: TMAP은 **계단(turnType 127/facilityType 16)을 반환하지 않음**(전국 공통, 확실한 계단 8곳 호출에도 미검출). → 대전에서도 계단 유무 태깅은 **OSM `highway=steps`·`incline`, 도로대장 종단경사+DEM** 보조 필요.

### 🤫 조용함 — 가장 얇은 풀 (최대 보완 대상)

- 국가소음 도로지점정보 CSV에서 **대전은 단 6개 지점**(상·하반기 12행) [실측]:

| 측정지점 | 주소 | 야간 등가소음(Leq) |
|---|---|:---:|
| 태평양화학 앞 | 대덕구 대화동 | **23.0 dB**(매우 조용) |
| 갤러리아백화점 앞 | 중구 대흥2동 | 33.8 dB |
| 삼부아파트 앞 | 중구 태평동 | 35.8 dB |
| 충남대부속병원 앞 | 중구 대사동 | 37.6 dB |
| 호텔리베라 앞 | 유성구 봉명동 | 39.8 dB |
| 한국타이어 앞 | 대덕구 목상동 | **48.4 dB**(시끄러움) |

- 문제 2가지: **(a) 6개는 도시 전역 보간에 절대 부족, (b) 원본에 위경도 없음(주소 텍스트만).**
- 보완: ① 주소 **지오코딩**으로 6점 좌표화 ② **도로교통소음_격자**(59394979)로 격자 소음 확보 ③ 대전 야간 무장애 소음지도(지자체) 병용.

## 3. 대전 MVP 종합 판정

| 목적 | 즉시 가용 | 활용 조건 | 판정 |
|---|:---:|---|:---:|
| 방범/조명 | ✅ 가로등 4.3만 + CCTV 다층 | 그대로 사용 | **강점** |
| 보도/보행 | ✅ 표준데이터 + TMAP | 표준데이터 활용신청 1건 | **강점** |
| 자연친화 | ✅ 둘레산길 12구간 | GPX 사용, POI는 502 복구 후 | **강점(차별화)** |
| 관절보호 | △ turnType | 계단 보조데이터 필요 | 보완 |
| 조용함 | △ 6지점 | 지오코딩 + 격자소음 | **최대 보완** |

**결론**: 대전 단일 MVP는 **"안전(방범)·자연친화"를 전면에 내세우는 구성이 데이터적으로 가장 튼튼하다.** 4대 목적 중 3개(방범·보도·자연)는 대전 데이터만으로 즉시 코스 스코어링이 가능하고, 나머지 2개(관절·조용)는 보조 데이터로 메우는 로드맵이 명확하다. 소음이 얇은 점만 초기 마케팅에서 과대포장하지 않으면 된다.

## 4. 대전 착수 액션 (우선순위)

1. **[방범]** 대전 가로등(15110054) + 방범용 CCTV(15109459) 다운로드 → 야간 안전 레이어 즉시 구축.
2. **[보도]** 전국보행자전용도로 표준데이터에 **대전 대상 활용신청** → 전용 serviceKey 발급(보도폭·CCTV·보안등·보차분리 한 번에).
3. **[자연]** 대전둘레산길 GPX 12구간 확보 → 자연친화 코스 시드. 국가숲길 POI API 502 복구 모니터링(제공기관 1566-0025).
4. **[조용]** 대전 6개 소음지점 지오코딩 + 도로교통소음_격자(59394979) 확보로 밀도 보강.
5. **[관절]** OSM `highway=steps`/`incline` 대전 영역 추출로 TMAP 계단 공백 메우기.
