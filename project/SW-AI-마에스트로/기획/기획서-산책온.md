# 산책온 프로젝트 기준서

> 2026-06-20 멘토 피드백 반영 기준서입니다.
> 

> 핵심 방향은 **코스 선택 부담 완화 + 추천 이유 제공 + 부담 없는 피드백/회고**입니다.
> 

> 미션 기능은 제외하고, 사용자가 산책을 시작하기 쉽게 만드는 위치 기반 산책 경험에 집중합니다.
> 

---

## 1. 프로젝트 최종 주제

### 주제명

> 지금 위치에서 바로 시작하는 AI 기반 맞춤형 산책 코스 추천 서비스
> 

### 서비스명

> 산책온
> 

### 한 줄 정의

> 산책온은 사용자의 현재 위치와 산책 조건을 기반으로 바로 시작 가능한 코스 후보와 추천 이유를 제공하고, 산책 후 간단한 피드백과 메모를 AI 회고 및 코스 품질 정보로 구조화하는 위치 기반 산책 서비스입니다.
> 

---

## 2. 핵심 포지셔닝

산책온은 **러닝 기록 앱, 리워드 앱, 등산·트레킹 앱**이 아닙니다.  

기존 앱이 거리, 시간, 페이스, 기록, 리워드 등 성과 관리 중심의 동기를 제공한다면, 산책온은 가볍게 산책하고 싶은 사용자가 **현재 위치에서 어디를 걸을지 쉽게 선택하고, 왜 이 코스가 적합한지 이해하며, 산책 후 경험을 회고**하도록 돕습니다.

```
산책 전: 어디를 걸을지 선택하기 어렵다
→ 산책온: 현재 위치·시간·목적·분위기 기반 코스 후보와 추천 이유 제공
→ 산책 중: 주요 지점 기반으로 산책 진행, 부담 없는 피드백 또는 한 줄 메모
→ 산책 후: AI 회고와 코스 태그화
→ 다음 추천: 선택형 피드백을 기반으로 품질 개선 참고
```

---

## 3. 해결하려는 문제

- 산책은 쉽게 시작할 수 있는 일상 활동이지만, 사용자는 산책을 시작하기 전 현재 위치에서 어떤 코스를 걸어야 할지 직접 선택해야 합니다.
- 산책 코스 선택에는 가능 시간, 목적, 선호 분위기, 예상 거리, 보행환경 등 여러 요소가 함께 고려되어야 하므로 단순한 최단 경로나 가까운 길 안내만으로는 충분하지 않습니다.
- 기존 지도 서비스는 목적지 이동 경로 안내에 강점이 있고, 기존 걷기·러닝·트레킹 서비스는 기록·성과·리워드 중심인 경우가 많아 가벼운 산책 목적에 맞는 코스 선택 경험은 제한적입니다.
- 산책온은 현재 위치와 산책 조건을 기반으로 바로 시작 가능한 코스 후보와 추천 이유를 제공하고, 산책 후 간단한 피드백과 회고를 통해 산책 경험을 정리하는 서비스를 제안합니다.

---

## 4. 산책온의 동기 제공 방식

멘토 피드백 기준으로 미션은 제외합니다. 산책온의 동기는 **강제 수행 과제**가 아니라 다음 요소에서 나옵니다.

| 동기 요소 | 설명 |
| --- | --- |
| 추천 이유 | 왜 이 코스가 현재 시간·거리·분위기 조건에 맞는지 설명 |
| 주요 지점 안내 | 출발지, 경유지, 반환점, 도착점을 보여주어 어디로 걸을지 명확화 |
| 부담 없는 피드백 | 산책 후 선택형 피드백과 한 줄 메모만으로 경험 기록 가능 |
| AI 회고 | 산책 기록과 피드백을 짧은 회고로 정리해 다음 산책 동기 제공 |

## 5. MVP 범위

### MVP에서 하는 것

- 현재 위치 기반 산책 조건 입력
- 공공데이터·지도 API 기반 주변 산책로·공원·보행로·주요 지점 조회
- 사용자 조건 기반 코스 후보 구성·추천
- 추천 이유 생성
- 주요 지점 기반 산책 진행
- 선택형 피드백 중심 기록, 한 줄 메모 보조
- AI 산책 회고 생성
- 피드백 태그화 및 관리자 검수 후보 제공

### MVP에서 하지 않는 것

- 체크포인트 미션 수행
- AI Agent가 전국 단위 코스를 완전 자동 생성
- 사용자 자유 메모만으로 추천 품질을 자동 개선
- AI가 안전 여부를 최종 판단해 자동 공개
- 친구 경쟁, 랭킹, 리워드, 캐릭터/아이템 기능
- 전국 서비스 지원

### MVP 검증 지역

> 초기 MVP는 대전/부산 등 제한 지역에서 검증합니다.
> 

> 공공데이터와 지도 API의 정확성은 현장 테스트와 선택형 피드백으로 보완합니다.
>

## 6. 피드백 설계 기준

자유 메모는 보조 수단이며, 기본은 선택형 피드백입니다.

| 피드백 항목 | 예시 | 활용 |
| --- | --- | --- |
| 분위기 | 조용함 / 활기참 / 자연친화적 / 도심형 | 코스 태그 |
| 길 상태 | 걷기 좋음 / 경사 있음 / 좁음 / 노면 불편 | 보행환경 태그 |
| 혼잡도 | 한적함 / 보통 / 혼잡함 | 시간대별 추천 참고 |
| 일시 이슈 | 공사 중 / 통제 구간 / 소음 / 악취 | 유효기간 기반 주의사항 |
| 만족도 | 좋음 / 보통 / 아쉬움 | 추천 품질 지표 |
| 한 줄 메모 | 필요 시 자유 입력 | 회고 생성 및 태그 보조 |

### 일시 이슈 처리

공사·통제 등 일시적 피드백은 등록 시점과 유효기간을 함께 저장하고, 기간이 지나면 추천 반영 가중치를 낮춥니다.

## 7. AI 활용 기준

AI는 서비스의 모든 판단을 대체하지 않습니다. AI 역할은 다음으로 제한합니다.

| AI 역할 | 설명 |
| --- | --- |
| 조건 해석 | 시간, 거리, 분위기, 난이도 조건을 추천 기준으로 변환 |
| 코스 후보 구성 보조 | 공공데이터·지도 API로 조회한 지점과 경로를 조건에 맞게 조합·정렬 |
| 추천 이유 생성 | 왜 이 코스가 적합한지 자연어로 설명 |
| 산책 회고 생성 | 산책 기록, 선택형 피드백, 한 줄 메모를 바탕으로 회고 생성 |
| 피드백 태그화 | 선택형 피드백과 메모를 분위기·길 상태·주의사항 태그로 구조화 |

## 8. 핵심 사용자 흐름

```
앱 실행
→ 현재 위치 확인
→ 산책 조건 선택
→ 코스 후보 및 추천 이유 확인
→ 코스 선택
→ 주요 지점 기반 산책 진행
→ 산책 완료
→ 선택형 피드백 작성
→ 필요 시 한 줄 메모 추가
→ AI 산책 회고 확인
→ 피드백 태그 기반 코스 품질 관리
```
