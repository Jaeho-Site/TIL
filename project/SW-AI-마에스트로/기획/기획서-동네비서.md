# 기획서 — 동네비서: 인앱 웹뷰 기반 로컬 라이프 AI 에이전트

> 소프트웨어 마에스트로 팀 프로젝트 기획서
---

## 0. 한 줄 요약

> **모바일 앱 인앱 웹뷰 안에서 동작하는 멀티모달 AI 에이전트.** 사용자가 "이거 팔고 싶어 / 이런 동네 서비스 찾아줘"라고 말하면, 에이전트가 **여러 도구(Tool)를 단계적으로 호출**해 사진 분석 → 시세 검색 → 글 초안 작성 → 등록까지 수행하고, **구매자-판매자 실시간 거래 채팅**에서 답변 추천·요약·사기 탐지를 보조한다.

---

## 2. 서비스 개요

### 2.1 문제 정의
동네 중고거래·로컬 서비스 이용 시 사용자는 **(1) 무엇을 얼마에 어떻게 팔/구할지 막막하고, (2) 거래 채팅에서 흥정·일정 조율·사기 판단에 피로**를 느낀다.

### 2.2 솔루션
앱 안의 **AI 에이전트**가 사용자의 자연어 요청을 받아 스스로 도구를 골라 일을 처리한다.

- **판매 플로우(에이전트 주도)**: 물건 사진 촬영 → 비전 분석으로 상품 식별 → 시세 검색 도구 호출 → 적정가 제안 → 판매글 초안 자동 작성 → 사용자 확인 후 등록
- **탐색 플로우(에이전트 주도)**: "우리 동네에서 30만원 이하 입식 책상" → 검색·필터·지도 도구를 조합해 후보 취합 → 비교표 생성
- **거래 채팅(에이전트 보조)**: 구매자-판매자 실시간 채팅에 에이전트가 답변 추천, 대화 요약, 위험 신호(외부 송금 유도 등) 탐지를 실시간 제공

### 2.3 사용자 가치
- 초보자도 **말 한마디로** 등록/탐색/흥정을 끝낸다
- 에이전트가 **무슨 일을 어떤 순서로 하는지 투명하게** 보여줘 신뢰 형성

## 4. 기획요구사항(`기획요구사항.md`) 5대 목표 매핑

### 4.1 ✅ 목표1 — 복잡한 상태관리
- **Zustand 전역 스토어 분리 설계**: `chatStore`(대화), `agentRunStore`(에이전트 실행 상태머신), `dealStore`(실시간 거래방), `uiStore`
- **서버 상태 vs 클라이언트 상태 분리**: 서버 데이터(매물·시세·메시지)는 **TanStack Query**, 클라이언트 임시 상태(초안·입력·낙관적 업데이트)는 Zustand
- **Agent 실행 상태머신**: `idle → planning → tool_running(toolName) → streaming → done/error` 를 명시적 상태로 모델링하고 UI에 1:1 매핑
- **실시간 이벤트 상태**: WebSocket(또는 SSE) 메시지를 받아 거래방 상태를 reduce
- **다중 사용자 동기화**: 구매자·판매자·에이전트 3주체의 메시지/타이핑/읽음 상태 동기화, 충돌 해소 규칙 설계
- **캐싱 전략**: 시세·매물 검색 결과 캐시 키 설계, staleTime/gcTime 튜닝, 동일 도구 재호출 시 캐시 재사용

### 4.2 ✅ 목표2 — WebView / 모바일 연동
- **인앱 웹뷰** 전제로 설계 (Android/iOS WebView)
- **Native Bridge**: `window.AppBridge` 추상화 레이어를 직접 만들어 카메라/파일/푸시/햅틱/안전영역(safe-area) 호출
- **카메라 접근**: 상품 촬영 → 비전 분석 입력
- **음성 입력(선택)**: 음성 → 텍스트로 에이전트 요청
- **Push Notification**: 에이전트 장시간 작업 완료 / 새 거래 메시지 알림
- **Deep Link**: 푸시 탭 → 특정 거래방·에이전트 작업 결과로 직접 진입
- **웹뷰 특수 문제 해결 스토리**: 키보드 올라올 때 입력창 가림, iOS 바운스 스크롤, 토큰 주입(Native→Web), 뒤로가기 제어 등 → **핵심 기술 포인트**

> 브릿지가 없는 일반 브라우저에서는 폴백(웹 카메라 API/웹 푸시)로 동작하게 추상화 → "환경 분기 설계" 경험.

### 4.3 ✅ 목표3 — CS 연결 성능 최적화
- **리스트 가상화**: 매물 피드·긴 대화 기록을 `@tanstack/virtual`로 가상화 → 수천 개 항목에서 DOM 노드 최소화
- **메모이제이션**: 채팅 메시지 컴포넌트 `React.memo` + 안정적 key, 스트리밍 중 리렌더 범위 격리
- **코드 스플리팅 / Lazy Loading**: 도구별 결과 패널(지도·비교표·이미지뷰어)을 동적 import
- **이미지 최적화**: 상품 사진 썸네일/원본 분리, lazy load, blur placeholder, 포맷 최적화
- **네트워크 최적화**: 스트리밍(SSE) 청크 처리, 요청 debounce/취소(AbortController), 재시도 백오프
- **Web Worker**: 대화/매물 데이터 파싱·검색 인덱싱 등 무거운 연산을 메인 스레드에서 분리
- **측정→개선 루프**: Lighthouse / React Profiler / `web-vitals`로 **before/after 수치화** → 개선 서사 핵심

### 4.4 ✅ 목표4 — Agent 기반 서비스 (단순 API 호출 X)
- 에이전트가 **도구를 선택·순차/병렬 호출**하는 멀티스텝 구조
- 예시 도구 세트: `analyzeImage`(비전), `searchMarketPrice`(시세), `searchListings`(매물검색), `getMapInfo`(지도/거리), `draftListing`(글 작성), `detectRisk`(사기 신호)
- 사용자 요청 분석 → 계획 수립 → 도구 호출 → 결과 취합 → 최종 응답의 **루프**
- (AI 서브 담당으로서) 프론트는 이 과정을 **이벤트 스트림으로 받아 UI에 투명하게 반영**하는 데 집중

### 4.5 ✅ 목표5 — AI UX / 에이전트 인터페이스
- **Chat UI + Streaming Response**: 토큰 단위 스트리밍 렌더링
- **에이전트 진행 상태 시각화**: "사진 분석 중 → 시세 검색 중(3건 조회) → 초안 작성 중" 타임라인/스텝퍼
- **Tool 실행 과정 표시**: 각 도구 호출의 입력·결과를 접이식 카드로 노출 (신뢰·투명성)
- **실시간 피드백**: 도구 중간 결과를 점진적으로 표시, 사용자가 중간 개입(중단·수정) 가능
- **긴 응답 최적화**: 가상화 + 점진적 마크다운 렌더링
- **대화 기록 관리**: 세션 저장/복구, 작업별 스레드 분리
- **Agent Workflow 시각화**: 어떤 도구가 왜 호출됐는지 보이는 워크플로우 뷰

| 분류 | 선택 | 이유 |

## 6. 상태 아키텍처 설계 (핵심 차별화)

```
[서버 상태 — TanStack Query]        [클라이언트 상태 — Zustand]
 - 매물/시세 검색 결과 (캐시 키 설계)   - chatStore: 입력·스트리밍 버퍼·스레드
 - 거래 메시지 히스토리                - agentRunStore: 실행 상태머신(현재 step/tool)
 - 사용자/동네 메타                    - dealStore: 실시간 거래방(참여자/타이핑/읽음)
        │                                   - uiStore: 패널·모달·바텀시트
        └──────── 동기화 경계 ───────────────┘
                    ▲
        [실시간 레이어 — WS/SSE 이벤트 → reducer → 두 영역에 반영]
```

**에이전트 실행 상태머신**
```
idle ──user msg──▶ planning ──tool 선택──▶ tool_running(tool, args)
   ▲                                          │ 결과 stream
   │                                          ▼
 done ◀── 최종 응답 streaming ◀── aggregating ◀── (도구 더 필요? 반복)
   └─────────────── error/cancel(사용자 중단) ◀──────┘
```
> 이 상태머신을 **UI 타임라인과 1:1로 매핑**하는 것이 "AI UX" 핵심 구현 포인트.

## 7. 성능 최적화 — 측정 기반 개선 계획

각 항목은 **before/after 지표**를 남겨 개선 근거로 사용한다.

| 영역 | 측정 지표 | 개선 기법 | 기대 서사 |
|------|----------|----------|----------|
| 첫 진입 | LCP, TTI | SSR + 코드 스플리팅 | "웹뷰 초기 로딩 N초 → M초" |
| 매물 피드 | 스크롤 FPS, DOM 노드 수 | 리스트 가상화 + 이미지 lazy | "1만건에서 끊김 제거" |
| 채팅 스트리밍 | 리렌더 횟수 | 메모이제이션 + 렌더 격리 | "메시지당 리렌더 X→1" |
| 무거운 연산 | 메인스레드 블로킹 시간 | Web Worker | "입력 지연 해소" |
| 네트워크 | 요청 수/취소 | 캐싱 + AbortController | "중복 호출 제거" |

## 10. 핵심 기술 도전 과제

이 프로젝트 하나로 다음 기술 경험을 깊게 확보할 수 있다.

- **AI UX 전달** → 에이전트 진행/도구 시각화
- **AI 개발 도구(Claude Code/Cursor) 활용** → 개발 과정 자체를 기록
